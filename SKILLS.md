# Aurevo.BE — Skills & Standards Reference

Living document. Update whenever a new module, pattern, or convention is established.

## Architecture

**Modular Monolith** — Domain-driven modules with MVC layers inside each module.
Overall: `Aurevo.UI → Aurevo.BE → Supabase (local Docker)`

```
Aurevo.BE/src/
├── app/
│   ├── modules/{domain}/
│   │   ├── {domain}.schema.ts      ← Zod validation schemas
│   │   ├── {domain}.service.ts     ← Drizzle queries + business logic
│   │   ├── {domain}.controller.ts  ← Express route handlers
│   │   └── {domain}.routes.ts      ← Router + middleware wiring
│   ├── middlewares/
│   │   ├── auth.ts                 ← authenticate, optionalAuth, requireAdmin
│   │   ├── validateRequest.ts      ← Zod validate() middleware
│   │   ├── rateLimiter.ts          ← Tiered rate limiters
│   │   └── globalErrorHandler.ts  ← Central error handler
│   ├── errors/
│   │   └── AppError.ts             ← AppError + typed subclasses
│   └── config/
│       ├── index.ts                ← Zod-validated env vars
│       └── swagger.ts              ← OpenAPI spec config
├── db/
│   ├── schema.ts                   ← Auto-generated by drizzle-kit introspect
│   ├── relations.ts                ← Auto-generated Drizzle relations
│   └── index.ts                    ← Drizzle client singleton
├── lib/
│   ├── supabase.ts                 ← Supabase admin client (service role)
│   └── storage.ts                  ← uploadFile / deleteFile / getPublicUrl
├── routes/
│   └── index.ts                    ← Aggregate all module routes
├── types/
│   └── express.d.ts                ← req.user / req.guestSessionId augmentation
├── app.ts                          ← Express app wiring
└── server.ts                       ← HTTP listener
```

## Tech Stack

| Concern | Tool |
|---------|------|
| ORM | Drizzle ORM (`drizzle-orm` + `postgres`) |
| DB | Local Supabase Docker — `postgresql://postgres:postgres@127.0.0.1:55322/postgres` |
| Auth | Supabase JWT (`SUPABASE_JWT_SECRET` + `jsonwebtoken`) |
| Storage | Browser → BE (multer) → Supabase Storage (service role key) |
| Validation | Zod on all request bodies + query params |
| Error handling | Global error handler + typed `AppError` subclasses |
| Rate limiting | `express-rate-limit` — tiered per route group |
| Docs | `swagger-jsdoc` + `swagger-ui-express` at `GET /api/docs` |
| TypeScript | `strict: true` |

## Local-Only Rule

**NEVER touch production DB.** All changes go to local Supabase Docker only.
- Local DB: `postgresql://postgres:postgres@127.0.0.1:54322/postgres`
- Local API: `http://127.0.0.1:54321`
- Production URL (`SYNC_PROD_DATABASE_URL` in Aurevo.UI) is read-only reference, never written to.

To re-introspect: `$env:DATABASE_URL="postgresql://postgres:postgres@127.0.0.1:54322/postgres"; npx drizzle-kit introspect`

## Response Shape Standard

**List endpoint:**
```json
{
  "success": true,
  "data": [...],
  "meta": {
    "pagination": {
      "page": 1, "limit": 12, "total": 45,
      "totalPages": 4, "hasNext": true, "hasPrev": false
    }
  }
}
```

**Single / Created / Deleted:**
```json
{ "success": true, "data": { "id": "..." } }          // 200 or 201
{ "success": true, "message": "Deleted successfully" } // 200 DELETE
```

**Error:**
```json
{ "success": false, "error": { "code": "NOT_FOUND", "message": "...", "details": {} } }
```

## Error Codes

| Code | HTTP | When |
|------|------|------|
| `NOT_FOUND` | 404 | Resource doesn't exist |
| `UNAUTHORIZED` | 401 | No/invalid JWT |
| `FORBIDDEN` | 403 | Valid JWT, wrong role |
| `VALIDATION_ERROR` | 400 | Zod schema failed |
| `CONFLICT` | 409 | Duplicate slug, already cancelled |
| `BUSINESS_RULE` | 422 | Business rule violation |
| `RATE_LIMIT` | 429 | Too many requests |
| `STORAGE_ERROR` | 500 | Supabase Storage failed |
| `INTERNAL_ERROR` | 500 | Unhandled server error |

## AppError Classes

```ts
throw new NotFoundError("Product");              // 404 NOT_FOUND
throw new ValidationError("msg", details);       // 400 VALIDATION_ERROR
throw new UnauthorizedError();                   // 401 UNAUTHORIZED
throw new ForbiddenError("Admin required");      // 403 FORBIDDEN
throw new ConflictError("Slug already exists");  // 409 CONFLICT
throw new BusinessRuleError("Can't delete");     // 422 BUSINESS_RULE
```

## Zod Validation Pattern

```ts
// {domain}.schema.ts
export const createProductSchema = z.object({
  body: z.object({ name: z.string().min(1), slug: z.string().regex(/^[a-z0-9-]+$/) }),
});
export const getProductsSchema = z.object({
  query: z.object({
    page: z.coerce.number().int().positive().default(1),
    limit: z.coerce.number().int().min(1).max(100).default(12),
    search: z.string().optional(),
  }),
});

// In routes file
router.get("/", publicLimiter, validate(getProductsSchema), getProducts);
router.post("/", authenticate, requireAdmin, validate(createProductSchema), createProduct);
```

## Composite Filter Pattern (list endpoints)

```ts
const conditions: SQL[] = [eq(products.isActive, true)];
if (filters.categoryId) conditions.push(eq(products.categoryId, filters.categoryId));
if (filters.search) conditions.push(ilike(products.name, `%${filters.search}%`));

const [rows, [{ total }]] = await Promise.all([
  db.select().from(products).where(and(...conditions)).limit(limit).offset(offset),
  db.select({ total: count() }).from(products).where(and(...conditions)),
]);
```

## Controller Pattern

```ts
export const getProducts = async (req: Request, res: Response, next: NextFunction): Promise<void> => {
  try {
    const result = await ProductService.getProducts(req.query as ProductFilters);
    res.status(200).json({ success: true, ...result });
  } catch (err) { next(err); }
};
```

## Auth Middleware

```ts
import { authenticate, optionalAuth, requireAdmin } from "../../middlewares";

router.get("/me", authenticate, getProfile);          // must be logged in
router.get("/", optionalAuth, getCart);               // guest or auth
router.delete("/:id", authenticate, requireAdmin, deleteProduct); // admin only
```

Role is read from `app_metadata.role` in the Supabase JWT. Admin role: `"admin"` or `"service_role"`.

## REST URL Conventions

- Plural nouns: `/api/products`, `/api/categories`
- Lowercase hyphenated: `/api/product-images`
- Nested: `/api/products/:id/variants`, `/api/products/:id/images`
- Actions: `/api/orders/:id/cancel`, `/api/cart/migrate`, `/api/courier/orders/:id/ship`

Standard GET list query params: `page`, `limit`, `search`, `sortBy`, `sortOrder`, plus domain-specific filters.

## Drizzle Schema (23 tables, 11 enums)

Key tables: `categories`, `brands`, `products`, `product_variants` (with `stock`/`reserved_stock`), `product_images`, `cart_items` (user_id or session_id), `orders` (sequential `order_number`, courier fields), `order_items`, `courier_tracking_events`, `inventory`, `inventory_movements`, `profiles`, `users`, `user_addresses`, `guest_sessions`, `meta_capi_sent`, `kb_chunks`, `conversations`, `messages`.

Enums: `product_gender`, `user_gender`, `order_status`, `payment_status`, `payment_method`, `fulfillment_status`, `movement_type`, `movement_reason`, `address_type`, `kb_source_type`, `chat_role`.

## Order Invoice PDF + Confirmation Email

- `src/lib/invoice-pdf.ts` — pdfkit buffer builder (mirrors `xlsx-export.ts` shape); Bengali font + SVG logo under `assets/`.
- `src/lib/email.ts` — Resend, no-op without `RESEND_API_KEY`; attaches invoice; PDF failure does not block send.
- Endpoint: `GET /api/orders/by-number/:orderNumber/invoice` (guest token / owner / admin).
- Order numbers: `ORD-` + 12-digit `order_number_seq` (fixed width for PDF layout).

---

## Courier (Steadfast)

- Module: `src/app/modules/courier/` · client: `src/lib/steadfast.ts`.
- Admin ship / refresh / balance; public track; Bearer webhook; internal poll cron.
- Optional env: `COURIER_API_KEY`, `COURIER_SECRET_KEY`, `COURIER_WEBHOOK_TOKEN`.
- Booking is never automatic — commits real money; no Steadfast cancel API.

## AI Chat Module (Phase 10, rebuilt as RAG)

Full retrieval-augmented generation pipeline, not a plain tool-use bot — see `docs/09-ai-chatbot-rag.md` for the complete architecture. Real Anthropic token streaming (`stream: true`) at `POST /api/chat`.
Tools: `search_knowledge` (semantic retrieval over `kb_chunks`/pgvector, optional `sourceType` filter), `get_product_details` (live stock/price by slug), `get_my_orders` (auth-gated only, hard-scoped server-side to `req.user.id`).
Rate limit: `chatLimiter`. Auth: `optionalAuth` (order lookup requires a real session). Model: `claude-haiku-4-5-20251001`.

Ingestion: `src/app/modules/knowledge/knowledge.service.ts` embeds products (auto, fire-and-forget on create/update) and `content/policies/*.md` (manual backfill via `pnpm ingest:knowledge`) into `kb_chunks` via Voyage AI.

SSE events: `{"text": "..."}` chunks, then `[DONE]`.
